This directory contains various technical documentations for the project, including the project charter, conceptual design documents, reference manuals, test plans, and more.

If a certain type of document have multiple files, they are organized into subdirectories named after the document type. Each document is named descriptively to indicate its content and purpose.

For example:

- `ProjectCharter.md`: Outlines the project's objectives, scope, stakeholders, and overall approach.
- `ConceptualDesign/`: Contains documents detailing the high-level architecture and design principles of the project.
- `ReferenceManuals/`: Includes user guides, API documentation, and other reference materials.
- `TestPlans/`: Contains test strategies, test cases, and testing procedures to ensure the quality of the project.
